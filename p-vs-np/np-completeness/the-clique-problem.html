The Clique Problem

I had never heard the word "clique" in a computer science context before I sat down to write this post. I knew it as a social word. A clique is a tight group where everyone knows everyone and nobody outside really gets let in. Schools have them. Offices have them.

When I found out that "clique" in graph theory means almost exactly the same thing, I was genuinely surprised. The mathematical definition is not a metaphor or a loose analogy. It is a direct formalization of the social idea. And the computational question it raises turns out to be one of the cleanest NP-Completeness proofs in this entire phase.

This post builds the Clique problem from the ground up, proves it is NP-Complete through a reduction from 3-SAT, and adds one more node to the reduction chain that started in Post 12.

A Quick Graph Recap​

Post 11 introduced graphs when we looked at Independent Set and Vertex Cover. A graph is just a set of nodes connected by edges. Nodes are dots. Edges are lines connecting pairs of dots.

In the social network framing I am going to use throughout this post, nodes are people and an edge between two people means they know each other directly.

Here is a small example with four people. I am going to write it as a connection list because that is cleaner than trying to draw it:

Nodes: A, B, C, D Edges: A -- B (A knows B) A -- C (A knows C) B -- C (B knows C) B -- D (B knows D) C -- D (C knows D)

A does not know D. There is no edge between them.

What a Clique Is​

A clique in a graph is a subset of nodes where every single pair in that subset is connected by an edge.

In social terms, a clique is a group where everyone personally knows everyone else. Not most of them. Every pair.

Take the example above. Is {A,B,C}\{A, B, C\}{A,B,C} a clique? Check every pair:

  • A knows B. Edge exists.
  • A knows C. Edge exists.
  • B knows C. Edge exists.

Every pair is connected. So yes, {A,B,C}\{A, B, C\}{A,B,C} is a clique of size 3.

Let me visualize just that triangle so the shape of a clique is clear: A / B---C

Every node connected to every other node. That is what a clique looks like.

Now ask whether {A,B,C,D}\{A, B, C, D\}{A,B,C,D} is a clique. Check every pair. A does not know D. No edge exists between them. So no, {A,B,C,D}\{A, B, C, D\}{A,B,C,D} is not a clique.

The number of nodes in the clique is called its size. A clique of size 3 is three people who all mutually know each other. A clique of size 10 is ten people who all mutually know each other, which means 45 separate pairwise connections all have to exist at once.

The Complete Subgraph

Mathematicians sometimes call a clique a complete subgraph. That just means taking a subset of nodes where every possible edge among them actually exists. Same idea, more formal language.

The Decision Problem​

Just like every other problem in this phase, we frame Clique as a yes/no question.

Clique problem. Given a graph GGG and a number kkk, does GGG contain a clique of size at least kkk?

Not "find the largest clique." Just "does one of this size exist?" Same underlying difficulty, but the yes/no framing is the one complexity theory works with.

Clique Is in NP​

First requirement for NP-Completeness is that the problem must be in NP\mathcal{NP}NP. That means there has to be a short certificate that can be verified fast.

For Clique the certificate is just the list of kkk nodes. If someone claims there is a clique of size kkk, they hand you that list.

Verification means checking every pair in the list and confirming an edge exists between them. The number of pairs in a group of kkk nodes is (k2)=k(k−1)2\binom{k}{2} = \frac{k(k-1)}{2}(2k​)=2k(k−1)​, which grows as O(k2)O(k^2)O(k2). Polynomial time. Fast.

Clique is in NP\mathcal{NP}NP.

Clique Is NP-Hard: The Reduction from 3-SAT​

This is the part I had to sit with for a while because the construction is not what I would have guessed. It is much cleaner than I expected and also a bit strange the first time you see it.

We need to show:

3-SAT≤PClique\text{3-SAT} \leq_P \text{Clique}3-SAT≤P​Clique

Given any 3-SAT formula, we convert it into a graph in polynomial time such that the formula is satisfiable if and only if the graph contains a clique of a specific size. Whoever can solve Clique can therefore solve 3-SAT. Since 3-SAT is NP-Complete, Clique must be NP-Hard.

The Construction​

Take any 3-SAT formula with kkk clauses. For the formula to be satisfied, at least one literal in each clause must be true. We need kkk simultaneously true literals, one from each clause, with no contradictions between them.

That is the key. We need a consistent selection of one true literal per clause.

Here is how the graph gets built.

Nodes. For each literal in each clause, create one node. A formula with kkk clauses and 3 literals per clause gives 3k3k3k nodes total. Each node carries a label (the literal it represents) and a tag (the clause it came from).

Edges. Connect two nodes with an edge if and only if both conditions hold:

  1. The two nodes come from different clauses.
  2. The two literals are not contradictory, meaning one is not the negation of the other.

No edges between nodes from the same clause. No edges between a literal and its negation from any clause.

The claim. The 3-SAT formula is satisfiable if and only if this graph contains a clique of size kkk.

Why This Works​

Direction 1. If the formula is satisfiable, the graph has a clique of size kkk.

Take a satisfying assignment. Pick one true literal from each clause. That gives kkk literals, one per clause. Look at the kkk nodes corresponding to them. Take any two of those nodes. They are from different clauses because we picked one per clause, so the first edge condition is met. Are they contradictory? No, because the satisfying assignment made both of them true simultaneously, and a literal and its negation cannot both be true at the same time. So the second condition is met too. Every pair has an edge. That is a clique of size kkk.

Direction 2. If the graph has a clique of size kkk, the formula is satisfiable.

Take a clique of size kkk. Two nodes from the same clause are never connected by construction. So the only way to have kkk mutually connected nodes is to have exactly one from each clause. Take the kkk literals those nodes represent. No two are contradictory because every pair in the clique is connected, and contradictory literals get no edge. So we can set all kkk literals to true simultaneously. That satisfies at least one literal in each clause, which satisfies the whole formula.

The Translation in Plain Words

A satisfying assignment is a consistent choice of one true literal per clause. A clique of size kkk is a consistent selection of one node per clause, where consistent means no contradictions. The construction turns literals into nodes and consistency into edges. A satisfying assignment and a clique of size kkk are the same object written in two different languages.

The Reduction Runs in Polynomial Time​

For a formula with kkk clauses and at most 3 literals per clause, we create at most 3k3k3k nodes. The number of potential edges is at most (3k2)\binom{3k}{2}(23k​), which is O(k2)O(k^2)O(k2). Checking each potential edge for the two conditions takes constant time.

Total construction time is O(k2)O(k^2)O(k2). Polynomial in the size of the formula.

The reduction is valid:

3-SAT≤PClique\text{3-SAT} \leq_P \text{Clique}3-SAT≤P​Clique

Since 3-SAT is NP-Complete and Clique is in NP\mathcal{NP}NP, Clique is NP-Complete.

Why Clique Feels Hard​

Searching for a clique of size kkk in a graph with nnn nodes by brute force means checking all possible subsets of kkk nodes and verifying each one. The number of such subsets is (nk)\binom{n}{k}(kn​).

  • Small k
  • Medium k
  • Large k

For k=3k = 3k=3 and n=50n = 50n=50, there are (503)=19,600\binom{50}{3} = 19{,}600(350​)=19,600 subsets to check. A computer handles this instantly. The problem feels trivial at this scale.

For k=10k = 10k=10 and n=50n = 50n=50, there are (5010)=10,272,278,170\binom{50}{10} = 10{,}272{,}278{,}170(1050​)=10,272,278,170 subsets. Over ten billion. The problem is already becoming uncomfortable.

For k=25k = 25k=25 and n=50n = 50n=50, there are (5025)≈1.26×1014\binom{50}{25} \approx 1.26 \times 10^{14}(2550​)≈1.26×1014 subsets. Hundreds of trillions. Post 6 showed what this growth regime actually means in practice. No hardware solves this in time.

Brute force is dead. No polynomial-time algorithm for finding the maximum clique is known. And now we understand that the difficulty is not accidental. Clique carries the same fundamental hardness as everything in NP\mathcal{NP}NP through the reduction chain.

The Thing That Surprised Me Writing This​

When I started working through this reduction, I kept expecting the hard part to be somewhere complicated. Some nested encoding, some trick that would take me a long time to follow.

The actual construction is almost shockingly direct. Literals become nodes. Consistency becomes edges. A satisfying assignment becomes a clique. Once you see it, you immediately see why it works. There is nothing hidden.

I think that directness is part of why this proof is one of the ones that stays with you.

Clique in the Reduction Chain​

Post 14 proved SAT≤P3-SAT\text{SAT} \leq_P \text{3-SAT}SAT≤P​3-SAT. This post proves 3-SAT≤PClique\text{3-SAT} \leq_P \text{Clique}3-SAT≤P​Clique. By transitivity:

Everything in NP→Cook-LevinSAT≤P3-SAT≤PClique\text{Everything in } \mathcal{NP} \xrightarrow{\text{Cook-Levin}} \text{SAT} \leq_P \text{3-SAT} \leq_P \text{Clique}Everything in NPCook-Levin​SAT≤P​3-SAT≤P​Clique

Any polynomial-time algorithm for Clique would propagate all the way back up and give us polynomial-time algorithms for everything in NP\mathcal{NP}NP.

What We Have Established

Clique is NP-Complete. It is in NP\mathcal{NP}NP because a proposed clique can be verified in polynomial time by checking all pairs. It is NP-Hard because 3-SAT reduces to it in polynomial time. Given a 3-SAT formula with kkk clauses, we build a graph where each literal becomes a node and edges connect non-contradictory literals from different clauses. The formula is satisfiable if and only if this graph contains a clique of size kkk. A clique is a consistent selection of one true literal per clause, which is exactly what a satisfying assignment is.
