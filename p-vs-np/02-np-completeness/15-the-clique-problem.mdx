# The Clique Problem

I want to be honest about this one.

When I first tried to understand the 3-SAT to Clique reduction, I read the standard explanation twice, nodded like I understood it, and moved on. Then I tried to explain it to myself from memory and completely blanked. I had the steps memorized but did not understand the *why* behind any of them.

The problem was that I was treating the reduction as a recipe to follow rather than an idea to understand. Once I stopped trying to follow the steps and started asking why each step exists, what problem each design choice is solving, it clicked.

That is how I want to explain it here. Not step by step, but idea by idea.

---

## What a Clique Is

A clique is a group of nodes in a graph where every single pair of nodes is connected by an edge.

That is the whole definition. If I pick four nodes and every possible pair among them has an edge, those four nodes form a clique of size 4.

The word comes from social networks. A clique of people is a group where everyone knows everyone else. Not just everyone knows one person in the group. Everyone knows every person in the group. Every possible pair is friends.

The question we are asking is the **Clique Decision Problem**: given a graph $G$ and a number $k$, does $G$ contain a clique of size $k$?

Checking a proposed clique is easy. If someone hands me a set of $k$ nodes and claims they form a clique, I just verify that every pair has an edge. That takes at most $k^2$ checks. Polynomial time. So Clique is in $\mathcal{NP}$.

Finding a clique is hard. There is no known polynomial-time algorithm to determine whether a clique of size $k$ exists in a general graph. And we are about to prove exactly why it is NP-Hard, by showing 3-SAT reduces to it.

---

## The Core Idea Behind the Reduction

Before I show you the construction, let me explain the intuition. This is the part that was missing for me every time I tried to learn this.

A satisfying assignment for a 3-SAT formula with $k$ clauses picks one literal per clause that is true. If the formula has 3 clauses, the assignment picks 3 literals, one per clause, that are all consistent with each other. Consistent here means: no two of them contradict each other. You cannot pick $x_1$ from one clause and $\neg x_1$ from another, because you cannot have $x_1$ be both true and false at the same time.

So a satisfying assignment is really just a set of $k$ consistent literals, one from each clause.

Now here is the reframe. What if we built a graph where:
- Each literal in the formula is a node
- Two nodes are connected if they are from different clauses and are not contradictory

Then a satisfying assignment, a set of $k$ consistent literals, one from each clause, would correspond exactly to a clique of size $k$ in that graph. Every pair in the set comes from different clauses (one per clause), and every pair is non-contradictory (they are all consistent). That is exactly what an edge means in this graph. So every pair has an edge. That is a clique.

And the reverse: a clique of size $k$ picks one node per clause (it cannot pick two from the same clause because same-clause nodes have no edges between them by design) and all pairs are consistent (because edges only exist between non-contradictory nodes). That is a satisfying assignment.

The formula is satisfiable if and only if the graph has a $k$-clique.

That is the whole reduction. Let me now build it concretely.

## The Construction

We will use a specific formula so everything is concrete. Here is our 3-SAT formula with 3 clauses ($k = 3$):

$$\phi = (x_1 \vee \neg x_2 \vee x_3) \wedge (\neg x_1 \vee x_2 \vee \neg x_3) \wedge (x_1 \vee x_2 \vee x_3)$$

**Step 1: Create one node per literal per clause.**

The formula has 3 clauses and 3 literals per clause, so we create $3 \times 3 = 9$ nodes.

| Node | Literal | Clause |
|------|---------|--------|
| 1 | $x_1$ | Clause 1 |
| 2 | $\neg x_2$ | Clause 1 |
| 3 | $x_3$ | Clause 1 |
| 4 | $\neg x_1$ | Clause 2 |
| 5 | $x_2$ | Clause 2 |
| 6 | $\neg x_3$ | Clause 2 |
| 7 | $x_1$ | Clause 3 |
| 8 | $x_2$ | Clause 3 |
| 9 | $x_3$ | Clause 3 |

**Step 2: Add edges between non-contradictory nodes from different clauses.**

Two nodes get an edge if and only if both conditions hold:
1. They come from **different clauses**
2. They are **not contradictory**; they do not have the same variable with opposite negation

Two nodes are contradictory if one is $x_i$ and the other is $\neg x_i$ for the same $i$. That is the only way to have a contradiction.

Let me work through every cross-clause pair systematically so nothing is mysterious.

:::note Why no edges within a clause?
Nodes from the same clause never get edges. The reason: a clique of size $k$ needs one node per clause, which means every pair in the clique must come from different clauses. If we allowed same-clause edges, a clique could pick two nodes from one clause and zero from another, which would not correspond to a valid satisfying assignment. Blocking same-clause edges enforces the "one per clause" rule automatically.
:::

## Reading the Clique as a Satisfying Assignment

Once you have the graph, finding a 3-clique gives you a satisfying assignment directly.

Try $x_1 = \text{true}$, $x_2 = \text{true}$, $x_3 = \text{true}$.

With this assignment:
- Clause 1: $x_1$ is true, so pick node 1 ($x_1$)
- Clause 2: $x_2$ is true, so pick node 5 ($x_2$)
- Clause 3: $x_1$ is true, so pick node 7 ($x_1$)

Nodes {1, 5, 7}. Do all three pairs have edges?

- Node 1 ($x_1$) and node 5 ($x_2$): different variables, edge exists ✓
- Node 1 ($x_1$) and node 7 ($x_1$): same literal, edge exists ✓
- Node 5 ($x_2$) and node 7 ($x_1$): different variables, edge exists ✓

All three pairs connected. That is a 3-clique.

The clique exists because the assignment is consistent: no two chosen literals contradict each other. And they cannot contradict each other, because that is exactly what the edge rule enforces. Every edge is a certificate of non-contradiction. Three mutually connected nodes are three mutually compatible choices.

:::tip The two-way correspondence
A satisfying assignment: pick one true literal per clause, and those nodes form a clique.

A clique of size $k$: one node per clause (same-clause pairs have no edges, so this is forced), all pairs non-contradictory (only non-contradictory pairs have edges), assign those literals true, and you have a satisfying assignment.

The correspondence goes in both directions, which is what makes this a valid reduction.
:::

## Why the Reduction Is Polynomial

For a formula with $k$ clauses and 3 literals per clause:
- We create $3k$ nodes. That is linear in the formula size.
- We check at most $(3k)^2 / 2$ pairs for edges. That is quadratic in $k$. Polynomial.

The whole conversion runs in polynomial time. And the output asks: does this graph have a clique of size $k$? Exactly one question, one number.

$$\text{3-SAT} \leq_P \text{Clique}$$

Since 3-SAT is NP-Complete and Clique is in $\mathcal{NP}$ (verifying a clique takes polynomial time: check that every pair has an edge), Clique is NP-Complete.

:::note What We Have Established
Clique is NP-Complete. It is in $\mathcal{NP}$ because a proposed clique can be verified in polynomial time. It is NP-Hard because 3-SAT reduces to it: given any 3-SAT formula, build a graph where literals are nodes and non-contradictory cross-clause pairs are edges. The formula is satisfiable if and only if the graph has a $k$-clique. The satisfying assignment and the clique are two descriptions of the same underlying structure: $k$ mutually consistent choices, one per clause.
:::
