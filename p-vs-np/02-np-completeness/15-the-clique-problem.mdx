# The Clique Problem

I had never heard the word "clique" in a computer science context before I sat down to write this post. I knew it as a social word. A clique is a tight group where everyone knows everyone and nobody outside really gets let in. Schools have them. Offices have them.

When I found out that "clique" in graph theory means almost exactly the same thing, I was genuinely surprised. The mathematical definition is not a metaphor or a loose analogy. It is a direct formalization of the social idea. And the computational question it raises turns out to be one of the cleanest NP-Completeness proofs in this entire phase.

This post builds the Clique problem from the ground up, proves it is NP-Complete through a reduction from 3-SAT, and adds one more node to the reduction chain that started in Post 12.

---

## A Quick Graph Recap

Post 11 introduced graphs when we looked at Independent Set and Vertex Cover. A graph is just a set of nodes connected by edges. Nodes are dots. Edges are lines connecting pairs of dots.

In the social network framing I am going to use throughout this post, nodes are people and an edge between two people means they know each other directly.

Here is a small example with four people. I am going to write it as a connection list because that is cleaner than trying to draw it:

Nodes: A, B, C, D
Edges:
A -- B  (A knows B)
A -- C  (A knows C)
B -- C  (B knows C)
B -- D  (B knows D)
C -- D  (C knows D)

A does not know D. There is no edge between them.

---

## What a Clique Is

A clique in a graph is a subset of nodes where every single pair in that subset is connected by an edge.

In social terms, a clique is a group where everyone personally knows everyone else. Not most of them. Every pair.

Take the example above. Is $\{A, B, C\}$ a clique? Check every pair:

- A knows B. Edge exists.
- A knows C. Edge exists.
- B knows C. Edge exists.

Every pair is connected. So yes, $\{A, B, C\}$ is a clique of size 3.

Let me visualize just that triangle so the shape of a clique is clear:
A
/ 
B---C

Every node connected to every other node. That is what a clique looks like.

Now ask whether $\{A, B, C, D\}$ is a clique. Check every pair. A does not know D. No edge exists between them. So no, $\{A, B, C, D\}$ is not a clique.

The number of nodes in the clique is called its size. A clique of size 3 is three people who all mutually know each other. A clique of size 10 is ten people who all mutually know each other, which means 45 separate pairwise connections all have to exist at once.

:::note The Complete Subgraph
Mathematicians sometimes call a clique a complete subgraph. That just means taking a subset of nodes where every possible edge among them actually exists. Same idea, more formal language.
:::

---

## The Decision Problem

Just like every other problem in this phase, we frame Clique as a yes/no question.

**Clique problem.** Given a graph $G$ and a number $k$, does $G$ contain a clique of size at least $k$?

Not "find the largest clique." Just "does one of this size exist?" Same underlying difficulty, but the yes/no framing is the one complexity theory works with.

---

## Clique Is in NP

First requirement for NP-Completeness is that the problem must be in $\mathcal{NP}$. That means there has to be a short certificate that can be verified fast.

For Clique the certificate is just the list of $k$ nodes. If someone claims there is a clique of size $k$, they hand you that list.

Verification means checking every pair in the list and confirming an edge exists between them. The number of pairs in a group of $k$ nodes is $\binom{k}{2} = \frac{k(k-1)}{2}$, which grows as $O(k^2)$. Polynomial time. Fast.

Clique is in $\mathcal{NP}$.

---

## Clique Is NP-Hard: The Reduction from 3-SAT

This is the part I had to sit with for a while because the construction is not what I would have guessed. It is much cleaner than I expected and also a bit strange the first time you see it.

We need to show:

$$\text{3-SAT} \leq_P \text{Clique}$$

Given any 3-SAT formula, we convert it into a graph in polynomial time such that the formula is satisfiable if and only if the graph contains a clique of a specific size. Whoever can solve Clique can therefore solve 3-SAT. Since 3-SAT is NP-Complete, Clique must be NP-Hard.

### The Construction

Take any 3-SAT formula with $k$ clauses. For the formula to be satisfied, at least one literal in each clause must be true. We need $k$ simultaneously true literals, one from each clause, with no contradictions between them.

That is the key. We need a consistent selection of one true literal per clause.

Here is how the graph gets built.

**Nodes.** For each literal in each clause, create one node. A formula with $k$ clauses and 3 literals per clause gives $3k$ nodes total. Each node carries a label (the literal it represents) and a tag (the clause it came from).

**Edges.** Connect two nodes with an edge if and only if both conditions hold:

1. The two nodes come from different clauses.
2. The two literals are not contradictory, meaning one is not the negation of the other.

No edges between nodes from the same clause. No edges between a literal and its negation from any clause.

**The claim.** The 3-SAT formula is satisfiable if and only if this graph contains a clique of size $k$.

### Why This Works

**Direction 1.** If the formula is satisfiable, the graph has a clique of size $k$.

Take a satisfying assignment. Pick one true literal from each clause. That gives $k$ literals, one per clause. Look at the $k$ nodes corresponding to them. Take any two of those nodes. They are from different clauses because we picked one per clause, so the first edge condition is met. Are they contradictory? No, because the satisfying assignment made both of them true simultaneously, and a literal and its negation cannot both be true at the same time. So the second condition is met too. Every pair has an edge. That is a clique of size $k$.

**Direction 2.** If the graph has a clique of size $k$, the formula is satisfiable.

Take a clique of size $k$. Two nodes from the same clause are never connected by construction. So the only way to have $k$ mutually connected nodes is to have exactly one from each clause. Take the $k$ literals those nodes represent. No two are contradictory because every pair in the clique is connected, and contradictory literals get no edge. So we can set all $k$ literals to true simultaneously. That satisfies at least one literal in each clause, which satisfies the whole formula.

:::tip The Translation in Plain Words
A satisfying assignment is a consistent choice of one true literal per clause. A clique of size $k$ is a consistent selection of one node per clause, where consistent means no contradictions. The construction turns literals into nodes and consistency into edges. A satisfying assignment and a clique of size $k$ are the same object written in two different languages.
:::

## A Worked Example

I want to trace through a real formula because I find it impossible to trust a proof until I have walked through the numbers myself.

Take this 3-SAT formula with 3 clauses ($k = 3$):

$$\phi = (x_1 \vee \neg x_2 \vee x_3) \wedge (\neg x_1 \vee x_2 \vee \neg x_3) \wedge (x_1 \vee x_2 \vee x_3)$$

**Step 1. Create the nodes.**

3 clauses, 3 literals each, so 9 nodes.

| Node | Literal | Clause |
|------|---------|--------|
| 1 | $x_1$ | 1 |
| 2 | $\neg x_2$ | 1 |
| 3 | $x_3$ | 1 |
| 4 | $\neg x_1$ | 2 |
| 5 | $x_2$ | 2 |
| 6 | $\neg x_3$ | 2 |
| 7 | $x_1$ | 3 |
| 8 | $x_2$ | 3 |
| 9 | $x_3$ | 3 |

**Step 2. Draw edges between clause 1 and clause 2.**

- Node 1 ($x_1$) and Node 4 ($\neg x_1$) are contradictory. No edge.
- Node 1 ($x_1$) and Node 5 ($x_2$) are not contradictory. Edge.
- Node 1 ($x_1$) and Node 6 ($\neg x_3$) are not contradictory. Edge.
- Node 2 ($\neg x_2$) and Node 4 ($\neg x_1$) are not contradictory. Edge.
- Node 2 ($\neg x_2$) and Node 5 ($x_2$) are contradictory. No edge.
- Node 2 ($\neg x_2$) and Node 6 ($\neg x_3$) are not contradictory. Edge.
- Node 3 ($x_3$) and Node 4 ($\neg x_1$) are not contradictory. Edge.
- Node 3 ($x_3$) and Node 5 ($x_2$) are not contradictory. Edge.
- Node 3 ($x_3$) and Node 6 ($\neg x_3$) are contradictory. No edge.

**Edges between clause 1 and clause 3.**

- Node 1 ($x_1$) and Node 7 ($x_1$) are the same literal. Edge.
- Node 1 ($x_1$) and Node 8 ($x_2$) are not contradictory. Edge.
- Node 1 ($x_1$) and Node 9 ($x_3$) are not contradictory. Edge.
- Node 2 ($\neg x_2$) and Node 7 ($x_1$) are not contradictory. Edge.
- Node 2 ($\neg x_2$) and Node 8 ($x_2$) are contradictory. No edge.
- Node 2 ($\neg x_2$) and Node 9 ($x_3$) are not contradictory. Edge.
- Node 3 ($x_3$) and Node 7 ($x_1$) are not contradictory. Edge.
- Node 3 ($x_3$) and Node 8 ($x_2$) are not contradictory. Edge.
- Node 3 ($x_3$) and Node 9 ($x_3$) are the same literal. Edge.

**Edges between clause 2 and clause 3.**

- Node 4 ($\neg x_1$) and Node 7 ($x_1$) are contradictory. No edge.
- Node 4 ($\neg x_1$) and Node 8 ($x_2$) are not contradictory. Edge.
- Node 4 ($\neg x_1$) and Node 9 ($x_3$) are not contradictory. Edge.
- Node 5 ($x_2$) and Node 7 ($x_1$) are not contradictory. Edge.
- Node 5 ($x_2$) and Node 8 ($x_2$) are the same literal. Edge.
- Node 5 ($x_2$) and Node 9 ($x_3$) are not contradictory. Edge.
- Node 6 ($\neg x_3$) and Node 7 ($x_1$) are not contradictory. Edge.
- Node 6 ($\neg x_3$) and Node 8 ($x_2$) are not contradictory. Edge.
- Node 6 ($\neg x_3$) and Node 9 ($x_3$) are contradictory. No edge.

**Step 3. Find the clique.**

I want a clique of size 3, one node per clause. Let me try Nodes 1, 5, 7 which are $x_1$ from clause 1, $x_2$ from clause 2, and $x_1$ from clause 3.

- Node 1 and Node 5 have an edge. Yes.
- Node 1 and Node 7 have an edge. Yes.
- Node 5 and Node 7 have an edge. Yes.

That is a valid clique of size 3.

**Step 4. Read back the satisfying assignment.**

From the clique: $x_1 = \text{true}$, $x_2 = \text{true}$. Set $x_3 = \text{false}$.

Check every clause:

- Clause 1: $(x_1 \vee \neg x_2 \vee x_3) = (\text{true} \vee \text{false} \vee \text{false}) = \text{true}$ ✓
- Clause 2: $(\neg x_1 \vee x_2 \vee \neg x_3) = (\text{false} \vee \text{true} \vee \text{true}) = \text{true}$ ✓
- Clause 3: $(x_1 \vee x_2 \vee x_3) = (\text{true} \vee \text{true} \vee \text{false}) = \text{true}$ ✓

All satisfied. The clique gave me the satisfying assignment directly.

## The Reduction Runs in Polynomial Time

For a formula with $k$ clauses and at most 3 literals per clause, we create at most $3k$ nodes. The number of potential edges is at most $\binom{3k}{2}$, which is $O(k^2)$. Checking each potential edge for the two conditions takes constant time.

Total construction time is $O(k^2)$. Polynomial in the size of the formula.

The reduction is valid:

$$\text{3-SAT} \leq_P \text{Clique}$$

Since 3-SAT is NP-Complete and Clique is in $\mathcal{NP}$, Clique is NP-Complete.

---

## Why Clique Feels Hard

Searching for a clique of size $k$ in a graph with $n$ nodes by brute force means checking all possible subsets of $k$ nodes and verifying each one. The number of such subsets is $\binom{n}{k}$.

<Tabs>
  <TabItem value="small" label="Small k" default>

For $k = 3$ and $n = 50$, there are $\binom{50}{3} = 19{,}600$ subsets to check. A computer handles this instantly. The problem feels trivial at this scale.

  </TabItem>
  <TabItem value="medium" label="Medium k">

For $k = 10$ and $n = 50$, there are $\binom{50}{10} = 10{,}272{,}278{,}170$ subsets. Over ten billion. The problem is already becoming uncomfortable.

  </TabItem>
  <TabItem value="large" label="Large k">

For $k = 25$ and $n = 50$, there are $\binom{50}{25} \approx 1.26 \times 10^{14}$ subsets. Hundreds of trillions. Post 6 showed what this growth regime actually means in practice. No hardware solves this in time.

  </TabItem>
</Tabs>

Brute force is dead. No polynomial-time algorithm for finding the maximum clique is known. And now we understand that the difficulty is not accidental. Clique carries the same fundamental hardness as everything in $\mathcal{NP}$ through the reduction chain.

## The Thing That Surprised Me Writing This

When I started working through this reduction, I kept expecting the hard part to be somewhere complicated. Some nested encoding, some trick that would take me a long time to follow.

The actual construction is almost shockingly direct. Literals become nodes. Consistency becomes edges. A satisfying assignment becomes a clique. Once you see it, you immediately see why it works. There is nothing hidden.

I think that directness is part of why this proof is one of the ones that stays with you.

## Clique in the Reduction Chain

Post 14 proved $\text{SAT} \leq_P \text{3-SAT}$. This post proves $\text{3-SAT} \leq_P \text{Clique}$. By transitivity:

$$\text{Everything in } \mathcal{NP} \xrightarrow{\text{Cook-Levin}} \text{SAT} \leq_P \text{3-SAT} \leq_P \text{Clique}$$

Any polynomial-time algorithm for Clique would propagate all the way back up and give us polynomial-time algorithms for everything in $\mathcal{NP}$.

:::note What We Have Established
Clique is NP-Complete. It is in $\mathcal{NP}$ because a proposed clique can be verified in polynomial time by checking all pairs. It is NP-Hard because 3-SAT reduces to it in polynomial time. Given a 3-SAT formula with $k$ clauses, we build a graph where each literal becomes a node and edges connect non-contradictory literals from different clauses. The formula is satisfiable if and only if this graph contains a clique of size $k$. A clique is a consistent selection of one true literal per clause, which is exactly what a satisfying assignment is.
:::
